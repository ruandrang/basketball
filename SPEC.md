# Basketball Club App — SPEC v0.1

> 개발/기획 공통 기준 문서 (MVP)

- 작성일: 2026-02-01 (KST)
- 목표: **농구 클럽 운영을 쉽게**(출첵/멤버관리/랜덤 팀/경기결과/승패통계/공유이미지) 하는 웹앱

## 0. 범위

### In (v0.1)
- 로그인/가입: 이메일+비번, SNS(구글/카카오/애플)
- 클럽 운영: 클럽 생성/설정, 멤버 관리(운영진/멤버)
- 출첵 관리: 모임 생성/출석 체크/마감
- 랜덤 팀 생성: 포지션 기반 + 팀당 센터 1명 이상 우선
- 경기 결과 기록: 게임 결과(승/패) + 개인 승/패 누적
- 공유: 랜덤 팀 결과/경기 결과 **이미지(PNG) 생성/다운로드**

> 벤치마크 참고(SportEasy): v0.1에서는 ‘팀/클럽 캘린더, 메시징, 자동 리마인더’는 우선 보류하지만 P1 후보로 유지.

### Out (v0.1에서는 보류)
- 개인 상세 스탯(득점/리바/어시 등)
- 고급 팀 밸런싱(5경기 이상 데이터 기반) — P1
- 대회 준비/대회 결과 관리 — **플랜(결제) 기반 확장 기능(P1)**
- 공유 링크(웹 공개/비공개) — P0 제외

## 4. 출첵(Attendance)

> SportEasy 인사이트: “이벤트 초대 + 자동 리마인더 + 실시간 참석 현황”이 팀 운영에서 가장 큰 체감 포인트.
> 우리 v0.1은 출첵만 먼저 구현하고, **자동 리마인더/푸시/이메일 알림은 P1로 확장**.

### 4.1 기본 단위
- 초기에는 **회차 개념 없음**
- **모임(날짜/이벤트) 1개 = 출첵 1개**

### 4.2 동작
- 운영진이 모임 생성
  - 예: `2026-02-01 정기모임`
- 멤버는 본인 상태를 참석/불참으로 변경
- 운영진은 전체 멤버 상태를 수정 가능
- 운영진이 마감(locked)

### 4.3 정책(초안)
- 마감 전: 운영진/멤버(본인) 수정 가능
- 마감 후: 운영진만 수정 가능(권장)

---

## 5. 랜덤 팀 생성(Random Team)

### 5.1 기본 팀 구성
- 기본은 **5:5**

### 5.2 포지션(고정)
- PG / SG / SF / PF / C

### 5.3 생성 규칙(v0.1)
우선순위:
1) **각 팀에 센터(C)가 최소 1명씩 배치되도록 시도**
2) 나머지 인원은 포지션 편중을 최소화하는 방향으로 분배

예외:
- 센터 수 < 팀 수:
  - 경고 표시
  - 옵션 제공
    - (A) 센터 룰 완화하고 진행
    - (B) 팀 수/인원 조정 안내

### 5.4 저장
- 팀 생성 결과는 히스토리로 저장(재조회 가능)

## 11. 벤치마크(SportEasy) 기반 P1 후보
SportEasy 팀/클럽 운영에서 유용해 보였던 스펙(우리 방향에 맞춰 변형):
- **팀/클럽 캘린더**: 경기/훈련/회식 등 이벤트 타입 + 필터
- **이벤트 초대 + 자동 리마인더**: 미응답/불참 방지, 푸시/이메일
- **팀 메시징**: 이벤트별 스레드(해당 모임 공지/질문) + 파일(이미지/PDF) 공유
- **태스크 할당**: 모임별 준비물/정리 담당 지정(운영진/멤버/자원봉사)
- **회비/비용 수금**: 연회비, 대회 참가비, 유니폼 구매 등(=결제/플랜과 결합)
- **경기 후 기능**(확장): MVP 투표, 후기/리포트(팀 문화 강화)
- **프로필 정보 공개 범위**: 연락처 등 민감정보 표시/숨김 (클럽별 권한)
